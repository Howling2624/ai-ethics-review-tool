设计原则

从现有界面抽象出的产品气质和使用规则。

  • 任务优先。首屏直接进入审查工作台，避免营销式介绍，重要操作固定在左侧或顶部。
  • 风险可见。冲突、警示、失败必须用稳定色彩和徽章表达，不依赖长文本。
  • 密度克制。表格、日志、统计卡保持紧凑，留足行高与边界线用于扫描。
  • 科技感来自细节。网格、发光和透明面板只作为背景语气，不能干扰阅读。
  • 本地工具感。按钮、输入框、文件上传和进度反馈要明确、朴素、可预测。
  • 明暗主题兼容。当前系统以暗色为主，但令牌已支持浅色主题扩展。

状态语言

建议保持颜色与语义的一一对应，降低误读成本。

成功 / 合规green: 下载成功、无冲突、已完成
处理中 / 警示amber: 运行中、涉及伦理、等待中
失败 / 风险red: 冲突、下载失败、分析错误
信息 / 链接blue: 报告链接、辅助高亮
未知 / 空态muted: 暂无结果、无法判断
焦点 / 输入green glow: focus ring, active tab

页面组合

当前应用的推荐布局：左侧命令面板，右侧运行反馈与结果表。

左侧：任务输入
  • 模式切换放在表单顶部。
  • 主按钮与导出按钮并排，移动端改为纵向。
  • 命令面板可在桌面端 sticky，保持可操作。
右侧：反馈与证据
  • 运行概览先给总数、完成数、风险数和状态。
  • 结果表格紧随其后，支持搜索和导出。
  • 日志作为可追踪证据，不抢占主要结果区。
